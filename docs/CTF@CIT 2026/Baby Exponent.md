- 核心突破点：这是典型的低指数 RSA。由于 `e = 3` 且明文很短，满足 `m^3 < n`，加密时没有发生模 `n` 回绕，所以 `c = m^3`，直接对 `c` 开整数立方根即可恢复明文。

## 入口与现象

题目只给了 `n`、`e`、`c`，没有附件、没有交互，也没有额外提示。这种题先看指数最值钱，因为很多“baby”级 RSA 题目会在指数、填充或明文范围上故意留洞。

这里 `e = 3` 非常小。如果明文本身又比较短，那么 RSA 加密公式

```text
c = m^e mod n
```

就可能退化成普通整数幂：

```text
c = m^3
```

原因是只要 `m^3 < n`，模运算就不会真正发挥作用。

## 分析过程

### 1. 判断是否存在低指数直接开根的条件

这题没有给任何填充信息，通常可以先假设明文就是 flag 字符串转成整数。CTF flag 往往长度不长，而给出的 `n` 非常大，所以 `m^3 < n` 是一个很合理的猜测。

验证方法也很直接：对 `c` 求整数立方根，看看得到的 `m` 是否满足 `m^3 == c`。如果等式成立，就说明猜测正确。

### 2. 对密文做整数立方根

用纯 Python 二分求整数立方根即可，不依赖额外库：

```python
n = 3975311104658158367804953186451876987828483822427305148759145730088615027289956528884778329789668637386484932183485546402292017850452360645365142100268336371204659887371551551598753305231985601246101574833959356250563521064956134365407699223
e = 3
c = 21208016443347524194488872231478291493949438339558450377152081476869432669496266457076405093626099218034592769060441274220970709748741037953818131469435699367735940032724483543045224740051080037

lo, hi = 0, 1
while hi ** 3 <= c:
    hi *= 2

while lo + 1 < hi:
    mid = (lo + hi) // 2
    if mid ** 3 <= c:
        lo = mid
    else:
        hi = mid

m = lo
print(m ** 3 == c)
print(m)
```

运行后 `m ** 3 == c` 为 `True`，说明这里确实没有发生模约简，密文就是明文整数的立方。

### 3. 把整数转回字节串

把 `m` 转成十六进制，再还原成字节串：

```python
m = 27680038115913774460198308299617031311839022658531567335298462333
hex_m = hex(m)[2:]
if len(hex_m) % 2:
    hex_m = "0" + hex_m

print(bytes.fromhex(hex_m))
```

输出结果为：

```text
b'CIT{sm4ll_3xp0n3nt_g0_brrr}'
```

## 利用过程

1. 观察到题目是裸 RSA 参数，优先检查 `e` 是否过小。
2. 发现 `e = 3`，考虑低指数攻击。
3. 假设明文较短，满足 `m^3 < n`，于是直接对 `c` 求整数立方根。
4. 验证得到的根满足 `m^3 == c`，确认无需分解 `n`。
5. 将 `m` 转为字节串，直接得到 flag。

## 总结

这题的核心就是 RSA 低指数场景下“明文太小”。很多人看到 RSA 会先想着分解 `n`，但这里完全没必要，因为漏洞不在模数，而在参数选择。只要 `e` 很小、明文又没有填充且范围不够大，`c = m^e mod n` 就会退化成普通幂运算，直接开根就能秒出明文。
